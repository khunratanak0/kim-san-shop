### 🚨 Critical Bugs & Hidden Traps
These are issues in your current code that will eventually break your app or cause headaches in production.

* **The 1MB Firestore Bomb (Base64 Images):** In your `AdminDashboard`, you are converting uploaded images into Base64 strings and saving them directly into the Firestore database (`imageUrl: base64`). 
    * **The Bug:** Firestore has a strict **1 Megabyte size limit per document**. If you upload a detailed hero image or a few heavy product images, your database saves will start failing out of nowhere.
    * **The Fix:** You need to implement **Firebase Storage**. Upload the image file to Firebase Storage, get the public download URL, and save *that URL* (a short string) to Firestore instead.
* **Next-Themes Hydration Mismatch:** You did a great job checking `if (mounted)` before rendering the theme toggle buttons, but you didn't wrap your main layout in a mounted check. If a user visits the site with a system preference of Dark Mode, but the server renders Light Mode first, Next.js will throw a hydration error in the console. 
* **Database Security Rules:** I can't see your Firebase console, but make sure your Firestore rules are locked down. The public should only have `read` access to the `products` and `settings` collections, and only authenticated admins should have `write` access. Otherwise, anyone with internet access could delete your catalog!

If you want this catalog to feel like a premium, professional storefront, here is what you are missing:

Right now, if someone clicks "Ask Seller," they probably just go to your Telegram profile. Make it do the heavy lifting for them!

**2. OpenGraph Meta Tags (The "Share" Experience)**
When people share your link on Telegram, WhatsApp, or Facebook, what does it look like? If you don't have OpenGraph (OG) tags, it looks like a boring, blank link.
* **Action:** Add Next.js metadata to your layout so that when a link is shared, it automatically pulls your `storeName`, `tagline`, and `heroImageUrl` into a beautiful preview card.

Catalogs are all about visuals. Right now, your images are constrained inside the `ProductCard`. 
* **Action:** Add a simple Lightbox or Modal. When a user taps a product image, it should expand to fill the screen so they can see the details.

Right now, you are fetching *all* products at once: `getDocs(collection(db, 'products'))`.
* This is totally fine for 20-50 products. But if you get up to 200+ products, fetching all those images and data on first load will make the site incredibly slow. 
* **Action:** Implement Firestore query limits (e.g., `limit(12)`) and a "Load More" button to fetch the next batch.

**5. Skeleton Loaders**
You currently have a nice spinning circle for the loading state. 
* **Action:** Upgrade this to "Skeleton" loading cards. Display 4-8 gray, pulsing placeholder cards while the data fetches from Firebase. It makes the site feel significantly faster and more modern.
